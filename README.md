# Cloud AI Storefront Builder

AI Storefront Builder is a TanStack Start application for creating storefront projects from natural-language prompts.

## Development

Install dependencies:

```bash
pnpm install
```

Run the dev server:

```bash
pnpm dev
```

Verify the project:

```bash
pnpm typecheck
pnpm lint
pnpm test
pnpm build
```

## Routing Convention

Builder routes use TanStack Start file-based routing in `src/routes`.

Current MVP routes:

- `/` — Home prompt flow.
- `/projects` — master-detail workspace with project list, virtual file explorer, message history, and composer.

Do not manually edit `src/routeTree.gen.ts`; it is generated by TanStack Router tooling.

## Storefront Builder Data Boundary

The UI uses service boundaries under `src/features/storefront-builder` and server functions under `src/server/functions`.

Current modules include:

- `project-service.ts` — create/list/load project workspaces.
- `message-service.ts` — list/send project messages with empty-message validation.
- `file-tree-service.ts` — load virtual project file trees and selected nodes.
- `mock-store.ts` — deterministic seed/default helpers only, not a client-only persistence layer.

Database-facing repository contracts live in `src/projects/repositories.ts`, with Drizzle schema additions in `src/db/schema.ts` for project messages and virtual file nodes.

## PWA Scope

PWA config belongs to generated storefront output, not the builder dashboard.

- Storefront PWA types/schema live in `src/storefront/types.ts` and `src/storefront/schema.ts`.
- PWA export boundaries live in `src/export/pwa-exporter.ts`.
- `pwa.enabled` controls manifest/service-worker generation.
- The service worker placeholder is limited to static storefront shell assets and avoids private API/user data caching.

## Design System

`DESIGN.md` is the source of truth for colors, typography, spacing, radius, and component treatment. Tokens are mapped into `tailwind.config.ts` and `app/styles/globals.css`. Do not modify `DESIGN.md` for feature implementation.
