# Cloud AI Projects

Cloud AI Projects is a TanStack Start application for creating AI-assisted projects from natural-language prompts.

## Development

Install dependencies:

```bash
pnpm install
```

Run the dev server:

```bash
pnpm dev
```

Verify the project:

```bash
pnpm typecheck
pnpm lint
pnpm test
pnpm build
```

## Routing Convention

Builder routes use TanStack Start file-based routing in `src/routes`.

Current MVP routes:

- `/` — Home prompt flow.
- `/projects` — master-detail workspace with project list, virtual file explorer, message history, and composer.

Do not manually edit `src/routeTree.gen.ts`; it is generated by TanStack Router tooling.

## Project Data Boundary

The UI uses service boundaries under `src/server/services` and server functions under `src/server/functions`.

Current modules include:

- `project-service.ts` — create/list/load project workspaces.
- `message-service.ts` — list/send project messages with empty-message validation.
- `file-tree-service.ts` — load persisted project file trees and selected nodes.

Database-facing repository contracts live in `src/shared/project-types.ts`, with Drizzle schema additions in `src/db/schema.ts` for projects, project messages, and project file nodes.

## PWA Scope

PWA fields are retained on project records for compatibility, but the current project creation flow does not create sample files, manifests, service workers, or placeholder assets.

## Design System

`DESIGN.md` is the source of truth for colors, typography, spacing, radius, and component treatment. Tokens are mapped into `tailwind.config.ts` and `app/styles/globals.css`. Do not modify `DESIGN.md` for feature implementation.

## Generated Storefront Workspaces

AI Agent generated storefront source is written outside the Builder Web runtime under `projects/{projectId}`. Builder routes and components must not import files from those generated workspaces directly; previews should use the project workspace or a dedicated sandbox/dev server. Agent patches are guarded to stay inside the matching project workspace, and `ProjectState` plus agent run history remain the source of truth for build/edit workflows.
