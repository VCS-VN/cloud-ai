# Implementation Plan: Axios HTTP Client Setup

**Branch**: `006-axios-http-client` | **Date**: 2026-05-12 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/006-axios-http-client/spec.md`

## Summary

Add a repeatable HTTP client baseline to generated project-detail workspaces: new projects should include a shared `src/services/http/client.ts`, safe environment key documentation, and axios `^1.16.0` in generated dependencies. The implementation will extend existing generated-source and code-agent prompt flows while avoiding preview runtime changes.

## Technical Context

**Language/Version**: TypeScript 6.0.3, React 19, TanStack Start/Router, Node runtime  
**Primary Dependencies**: axios `^1.16.0`, TanStack Start, TanStack Router, generated shadcn-style component stack  
**Storage**: Generated project files on disk plus project file-node metadata; no database schema change  
**Testing**: Vitest for source-generation behavior, TypeScript typecheck via `pnpm lint`  
**Target Platform**: Web application project generator and generated browser app  
**Project Type**: Full-stack web app with generated storefront workspaces  
**Performance Goals**: Project initialization adds only small static files; no measurable preview startup regression  
**Constraints**: Must use `@/` imports for generated cross-folder imports; must not edit preview lifecycle flow; `.env` secrets remain protected  
**Scale/Scope**: Applies to newly generated project-detail workspaces; existing preview behavior remains unchanged
