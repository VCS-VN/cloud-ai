CREATE THESE 5 DATA HOOKS (all under src/services/store/*). Every route/component consumes these — they do NOT import @/data/* directly. Each hook returns real API data when VITE_STORE_SLUG exists and a sample fallback (from @/data/products, @/data/categories) when it is missing, so consumers never branch on hasStoreSlug. All hooks import `{ apiClient } from '@/services/http/client'` and use @tanstack/react-query. Export a `hasStoreSlug` boolean (VITE_STORE_SLUG present) and an `isClientRuntime` guard (typeof window !== 'undefined') from use-store-detail.ts and reuse them.

src/services/store/use-store-detail.ts - useStoreDetail() hook. GET /api/v1/stores/:storeSlug for store detail via apiClient when hasStoreSlug; queryKey ['store-detail', slug]; enabled = hasStoreSlug. Sample fallback when slug missing returns `sampleStore` from @/data/sample-store. Returns { data: StoreDetail | undefined, isLoading, isError, error, refetch, isUsingSampleData }. Export `hasStoreSlug` (Boolean(import.meta.env.VITE_STORE_SLUG)), `isClientRuntime` (typeof window !== 'undefined'), and types StoreDetail = { id: string; name: string; slug?: string; setting?: StoreSetting } and StoreSetting = { currency?: string }.

src/services/store/use-products-list.ts - useProductsList({ storeId, query }) hook. MUST use useInfiniteQuery: GET /api/v1/products with params { limit: 12, page (1-indexed), storeId, query } via apiClient; queryKey ['products-list', storeId, query]; enabled = hasStoreSlug && Boolean(storeId); getNextPageParam returns next 1-indexed page only when (sum of items.length across loaded pages) < lastPage.total, else undefined. Response page shape { total: number, data: Product[] }. Returns { products: Product[] (pages.flatMap(p => p.data)), total, fetchNextPage, hasNextPage, isFetchingNextPage, isLoading, isError, error, refetch }. Sample fallback when slug missing returns the full `products` array from @/data/products with total = products.length and hasNextPage = false (optionally substring-filtered by query). Export types Product = { id: string; name: string; descriptions?: string; image?: string; images?: string[]; category?: { id: string; name: string }; price?: number; compareAtPrice?: number; defaultModel?: ProductModel; models?: ProductModel[] }, ProductModel = { id: string; name: string; price: number }, and ProductsList = { total: number; data: Product[] }. Prices are integer cents. The field is `descriptions` (plural); there is NO `description`.

src/services/store/use-product-detail.ts - useProductDetail(productId) hook. GET /api/v1/products/:productId with params { isGettingModels: true, isGettingDefaultModel: true } via apiClient; queryKey ['product-detail', productId]; enabled = hasStoreSlug && Boolean(productId). Response carries defaultModel and models[] alongside base Product fields. Returns { data: ProductDetail | undefined, isLoading, isError, error, refetch }. Sample fallback when slug missing finds the product by id in @/data/products. Export type ProductDetail = Product (reuse the Product type from use-products-list via import).

src/services/store/use-categories-list.ts - useCategoriesList({ storeId }) hook. GET /api/v1/categories with params { storeId } via apiClient; queryKey ['categories-list', storeId]; enabled = hasStoreSlug && Boolean(storeId). Response shape CategoriesList { total: number, data: Category[] }. Returns { categories: Category[], total, isLoading, isError, error, refetch }. Sample fallback when slug missing returns `categories` from @/data/categories. Export types Category = { id: string; name: string; storeId?: string } and CategoriesList = { total: number; data: Category[] }.

src/services/store/use-product-suggestions.ts - useProductSuggestions({ storeId, query }) hook. When VITE_STORE_SLUG exists AND storeId truthy AND query.trim().length > 0, call GET /api/v1/products/suggestions from the client runtime only with params { storeId, query } via @tanstack/react-query useQuery and apiClient.get<ProductSuggestionsList>('/api/v1/products/suggestions', { params }); response shape ProductSuggestionsList = { total: number, data: string[] }. queryKey ['product-suggestions', storeId, query.trim()]; enabled = hasStoreSlug && Boolean(storeId) && query.trim().length > 0; `isClientRuntime && ...` is allowed as a defensive guard when useful. When VITE_STORE_SLUG is missing OR query is empty, return a deterministic local sample list (filter product names from @/data/products by case-insensitive substring against query, dedupe, cap at 8) — mirror the sample-fallback pattern used by useProductsList. The hook returns { suggestions: string[], total: number, isLoading: boolean, isError: boolean, error: unknown, refetch: () => unknown }. Export ProductSuggestionsList type.
